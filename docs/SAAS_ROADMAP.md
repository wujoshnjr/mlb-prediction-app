# SaaS Roadmap

## Current Stage

The current product is a **MLB paper-trading research dashboard and market comparison evidence pipeline**.

It is not a real-money betting system. It does not place wagers, connect to sportsbook execution APIs, handle user funds, or guarantee profit.

## Target Stage

The long-term target is a **Sports Intelligence SaaS / B2B API platform** that provides auditable market intelligence, prediction evidence, CLV tracking, model governance, data-quality monitoring, and read-only analytics infrastructure.

The target product is not an automated wagering platform.

## Phase 1: Research Platform

Status: current stage.

Goals:

- Unify sample state.
- Keep prediction snapshots timestamped and auditable.
- Use finalized games as the only trusted outcome source.
- Track market baseline comparisons.
- Track closing-line value.
- Maintain paper-only governance.
- Build a credible dashboard for research and model review.

Current modules:

- Prediction snapshots
- Finalized outcome linking
- Sample state tracking
- Market close report
- CLV diagnostics
- Promotion gate
- Data contract validation
- Pipeline manifest
- World-class trading system control tower
- SaaS readiness tracking

Not allowed in this phase:

- Real-money betting
- Automated wagering
- Sportsbook execution
- User fund custody
- Commercial API access

## Phase 2: Paid Dashboard

Status: planned.

Goals:

- Add user login.
- Add subscription plans.
- Add dashboard permissions.
- Add CSV export.
- Add email summary.
- Add usage terms and risk disclosure.

Readiness requirements:

- Stable documentation.
- Clear risk policy.
- Sufficient OOS evidence.
- License review for all data sources.
- No automated wagering.

## Phase 3: B2B API

Status: planned.

Goals:

- API key authentication.
- Rate limiting.
- Usage tracking.
- OpenAPI documentation.
- Read-only predictions API.
- Market odds API.
- Data quality API.
- CLV report API.
- OOS report API.

Not allowed:

- Sportsbook execution.
- Automated bet placement.
- Fund movement.
- Profit guarantee.

## Phase 4: Enterprise / White-label

Status: planned.

Goals:

- Multi-tenant organizations.
- RBAC.
- Audit logs.
- White-label widgets.
- Webhooks.
- SLA documentation.
- Enterprise security package.
- Data licensing registry.
- Developer portal.

## Disabled Modules

The following modules are disabled by policy:

- real_money_betting
- automated_wagering
- sportsbook_execution
- user_fund_custody
- commercial_api_distribution_until_license_review

## Guiding Principle

Do not sell “guaranteed picks.” Build and sell verifiable sports intelligence infrastructure.
